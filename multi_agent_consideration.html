Multi-Agent Systems

When to Use Them

Most teams overestimate how much they need multiple agents.
One agent with good instructions often outperforms multiple agents.

What You Need to Know

One Agent
You ask Claude → It solves it → Done
Multiple Agents
Lead agent spawns specialists → Each handles part → Results combine
The cost: 3-10x more tokens, more complexity, more failure points
The rule: Only add agents when you're stuck on a specific problem.

Three Situations Where Multiple Agents Win

1. Information Overload
Early data clutters thinking. Separate agent extracts essentials. Main agent stays focused.
2. Parallel Work
Independent pieces run simultaneously. Larger information space covered than one agent can handle.
3. Too Many Tools
15+ tools confuse the AI. Specialist agents with focused toolsets pick correctly.

The Common Mistake

Don't: Divide by work phases
Agent 1 plans → Agent 2 builds → Agent 3 tests → Agent 4 reviews
Result: Handoff losses, coordination overhead, lost context
Do: Divide by context boundaries
One agent handles full feature with its tests. Different agent handles different feature.
Result: Minimal handoffs, focused context, better results

The Verification Agent Pattern

This one pattern works everywhere: Main agent does work → Separate agent verifies it works.

Why It Works
Verifier only needs to test (minimal context required). Doesn't need to understand design decisions. Clean separation of concerns.
Examples
Code: agent writes, verifier runs tests. Content: agent writes, verifier fact-checks. Compliance: agent creates, verifier checks rules.
Common Failure:
Verifier checks halfway and declares success. Fix: Be explicit. "Run full test suite. Report every failure. Only mark PASSED if 100% pass."

Decision Framework

Do I have a specific problem?
No → Use one agent
Is it ONE of these three?
Too much info cluttering thinking?
Independent work to parallelize?
More than 15 tools causing errors?
Yes to any → Multiple agents might help
No to all → One agent + better prompting
Before adding agents, try:
Better prompting • Better tools • Tool Search
Context compaction • Multiple separate chats

Key Takeaway

Start with one great agent. Add more only when you prove you need them.
Measure the impact: Single agent baseline vs. your approach. Did you hit one of the three situations? Is the quality jump worth 3-10x more tokens?
Most teams never need to get there.
