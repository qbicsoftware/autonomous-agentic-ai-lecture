Agent = LLM + Loop + Tools

  • An autonomous system that reasons, plans, and acts via external tools
  • It is not just a chatbot — it executes actions on your behalf
  • The core functionality is always the same, regardless of task complexity

Escape Gate
  • Task complete — the LLM signals it needs no more tools
  • Max iterations — a hard cap prevents infinite loops
  • Error / unrecoverable — execution fails and safety kicks in
  • User interrupts — explicit stop or cancellation

  • System Prompt: Instructions, personality, and available capabilities
  • Tool Registry: Catalog of functions the agent is allowed to invoke
  • Memory: Conversation history, file contents, and intermediate results

How Agents Talk to LLMs

  • System Prompt primes the model with identity, rules, and tool descriptions
  • Tool Schemas are injected into the prompt so the LLM "knows" what's available
  • Function Calling API constrains output to valid JSON matching a schema
  • The LLM probabilistically generates which tool to call — it is not a hard-coded rule
{ "name": "run_python", "description": "Execute Python code for data analysis", "parameters": { "type": "object", "properties": { "code": { "type": "string", "description": "Python code to run" } }, "required": ["code"] } }

Who Decides vs. Who Executes

REMOTE
LLM
  • Receives tool schemas inside the prompt
  • Probabilistically generates a tool call JSON
  • Constrained by API grammar to valid schemas
  • Never touches your files — only plans
↓
JSON tool_call
↓
LOCAL
Agent Runtime
  • Parses the JSON request
  • Executes the native function
  • Captures stdout / result
  • Runs on your machine

The LLM only plans. The agent acts in your environment.

Tools & Skills

🔧 Tool

  • Atomic capability
  • Schema + executor
  • Examples: bash, read_file, web_search

🧠 Skill

  • Curated expertise bundle
  • Tools + prompt templates + domain logic
  • Examples: GitHub CLI skill, Data analysis skill

MCP — Model Context Protocol

  • An open standard (pioneered by Anthropic) for connecting agents to external systems
  • Think of it as USB-C for AI capabilities
  • An MCP Server exposes resources, prompts, and tools
  • Any MCP Client (agent) can consume them dynamically

Why This Revolutionizes Coding Agents

  • Dynamic discovery: Agents adapt to your stack instead of relying on hardcoded scripts
  • Universal plug-ins: One protocol connects any data source or tool
  • For researchers: automate analysis, literature search, version control, and experiments
  • Decoupled architecture: Tool authors and agent builders work independently

The Big Picture

  • The core loop (Perceive → Reason → Act → Observe) is timeless
  • The power comes from better LLMs, richer tools, and standardized protocols
  • MCP turns isolated agents into extensible, general-purpose platforms
  • For your daily work: install an agent, connect MCP servers, automate workflows
